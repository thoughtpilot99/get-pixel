Section 1

What is Pixel?

Pixel is Metadata's AI-powered marketing agent that enables B2B marketers to plan, build, and launch multi-channel advertising campaigns through natural-language conversation. Instead of navigating the Metadata platform's menus and forms manually, users describe their campaign goals in plain English, and Pixel handles the end-to-end execution.

Key Value Propositions

ValueDescription
Conversational Campaign BuildingUsers describe what they want; Pixel builds it step by step
Multi-Channel OrchestrationA single conversation can produce campaigns across LinkedIn, Facebook, Instagram, and Google Ads simultaneously
AI-Powered CreativesGenerates brand-consistent ad images using the company's Brand DNA (colors, fonts, logo, messaging)
Intelligent Tool SelectionPixel dynamically selects the right platform tools based on the user's intent, using semantic search over 50+ available tools
Built on 10+ Years of Platform MaturityEvery action Pixel takes goes through the same battle-tested Metadata platform APIs used by thousands of marketers
Guardrailed ExecutionCredit validation, auto-pause safeguards, and human-in-the-loop campaign launching ensure no unintended spend

Section 2

How Pixel Works

Pixel operates as a ReAct (Reasoning + Acting) agent: it reads the user's message, reasons about what steps are needed, selects the appropriate tools, executes them, observes the results, and repeats until the task is complete.

Intent Classification

Each user message is classified into one of three types:

IntentDescriptionExample
QuestionInformation requests, analysis queries"What's the best channel for my budget?"
ActionTask execution requests"Create a lead gen campaign targeting CFOs"
ContinuationMulti-turn workflow progression"Now let's add Instagram as a channel"

Intelligent Tool Selection

Pixel does not load all 91 MCP server tools at once. Instead, it uses embedding-based semantic search to select the most relevant tools for each conversation turn:

  • Core Tools (10 tools) are always available — essential for any workflow (e.g., campaign search, brand kit, credit balance)
  • Contextual Tools are dynamically selected based on the user's query using cosine similarity against tool descriptions
  • Dependency Tools are automatically included when a selected tool requires another to complete its workflow
  • Denied Tools (36 tools) are permanently excluded from selection for security or controlled-rollout reasons
Section 3

The Six-Phase Workflow

When a user requests a campaign, Pixel follows a strictly sequential six-phase workflow. Each phase must complete before the next begins.

01

Planning

Pixel presents a strategic plan with recommended channels, targeting approach, and creative direction.

StepWhat Pixel Does
Credit checkValidates the user has sufficient credits to proceed
Market researchSearches the web for industry trends, competitor strategies, and channel best practices
Channel analysisCompares channel performance (LinkedIn vs Facebook vs Google vs Instagram) using historical benchmarks
Brand discoveryRetrieves or generates the company's Brand Kit (colors, fonts, logo, messaging framework)
Targeting analysisAnalyzes historical targeting patterns (industry x seniority x job function) for lead optimization
02

Audience Building

Identifies optimal industries, job titles, seniority levels, geographic locations and creates audiences.

StepWhat Pixel Does
Research criteriaIdentifies optimal industries, job titles, seniority levels, geographic locations
Create audiencesBuilds one or more audiences (firmographic, technographic, or native channel audiences)
Validate sizingConfirms audience sizes are viable for campaign delivery
Supported Audience Types: Firmographic (company attributes), Technographic (technology stack), LinkedIn Native Criteria (job titles, skills, company names), Facebook Native Criteria (demographics, interests, job titles)
03

Offer Creation

Builds Lead Gen forms or Landing Page offers per channel.

StepWhat Pixel Does
Privacy URL validationFinds and validates the company's privacy policy URL (required by ad channels)
Offer researchAnalyzes historical offer performance patterns
Create offersBuilds Lead Gen forms or Landing Page offers per channel
Business Rule: Lead Gen campaigns can ONLY use Lead Gen offers. Brand Awareness campaigns accept both types.
04

Creative & Ad Production

Generates branded ad images, uploads them, and creates ads with channel-specific copy.

StepWhat Pixel Does
Ad copy researchAnalyzes historical ad performance for optimal copy patterns
Creative generationGenerates branded ad images using AI + company Brand DNA
Image uploadUploads generated images to the Metadata creative library
Ad creationCreates ads with channel-specific copy, headlines, descriptions, and CTAs
05

Campaign Assembly

Assembles all components into a campaign with budget allocation.

StepWhat Pixel Does
Budget group creationCreates a budget group with the specified allocation and spending strategy
Campaign creationAssembles all components (channels, audiences, offers, ads, budget) into a campaign
Configuration displayShows the complete campaign configuration in the UI for user review
Business Rules: Campaign start date is always today (not a future date). Default campaign duration: 90 days. Budget group end date must extend beyond campaign end date.
06

Launch Presentation

The user reviews and decides when to go live.

StepWhat Pixel Does
Campaign displayRenders the full campaign card in the chat UI
User reviewUser can edit dates, budget, channels, and review all components
User-initiated launchThe user clicks the Launch button — Pixel never launches campaigns autonomously
Key Safety Principle: Campaign launching is always a human action. Pixel builds and configures; the user decides when to go live.

Audience Management Actions

  • Create new audiences with detailed targeting criteria
  • View existing audiences with full criteria details
  • Add audiences to campaigns (including across multiple channels)
  • Exclude audiences from campaigns (negative targeting)
  • Archive unused audiences to keep the library clean
  • Unarchive previously archived audiences for reuse

Audience Types Not Currently Available via Pixel

The following audience types exist on the Metadata platform but are temporarily disabled for Pixel:

  • Bombora Buyer Intent audiences (intent topic-based targeting)
  • G2 Dynamic audiences (buying stage-based targeting)
  • Retargeting audiences (website visitors, channel retargeting)
  • ABM Account Lists (account-based marketing lists)
  • Target Groups (LinkedIn native target groups with advanced criteria)
  • Segment-based audiences (insights-derived retargeting)
These are planned for future enablement as the tool set is progressively expanded.

Credit System & Budget Controls

Every Pixel conversation and campaign action consumes Pixel AI Credits, with multiple layers of budget protection.

Pixel AI Credits

ConceptDescription
Credit BalanceEach account has a credit plan with a total allocation
Consumption EventsActions like brand kit generation, creative generation, audience creation, campaign publishing, and lead enrichment each consume credits
Credit CheckPixel validates sufficient credits at the start of every workflow (Phase 1)
Credit ReportingPer-conversation credit consumption is tracked and reportable

Budget Controls & Limits

ControlDescription
Contract AllocationHard limit per billing cycle (quarterly, monthly, or annual)
Overage PercentageConfigurable percentage above allocation that's permitted (e.g., 20% overage on $10K = $12K max)
Unlimited Overage OptionCan be enabled per account to remove overage caps
Workspace SharingParent accounts hold limits; child accounts share the pool
Ad-Hoc CreditsPrepaid top-up credits for additional capacity

Auto-Pause Safeguard

Critical Safety Feature: When campaign ad spend exceeds the account's contract allocation for the current month, all launched campaigns are automatically paused to prevent further spend. This is a binary safeguard — there is no gradual throttling. Campaigns remain paused until manually resumed after budget adjustment.

Section 15

Safety Controls & Guardrails

Pixel is built with multiple layers of safety controls to protect users and prevent unintended actions.

Operational Safeguards

SafeguardDescription
Human-in-the-loop launchingPixel never launches campaigns autonomously; the user must click Launch
Credit validationCredit balance is checked before starting any workflow
Auto-pauseCampaigns are automatically paused when ad spend exceeds budget allocation
Asset naming conventionAll created assets include a 4-character UUID suffix to prevent naming conflicts
Privacy URL validationPrivacy policy URLs are validated before offer creation
Duplicate preventionPixel tracks the last 10 tool calls to prevent repeating identical actions
Loop detectionIf 3+ identical tool calls are detected in a 10-call window, Pixel stops and explains the issue
Tool call limitMaximum 15 tool calls per response to prevent runaway execution
Recursion limitMaximum 50 reasoning steps per conversation turn
Graceful stopUsers can stop Pixel mid-execution; it completes the current step and returns status

Tool Access Control

  • 36 tools are permanently denied — including account impersonation, advanced insights, and campaign launching
  • 10 core tools are always available — essential tools for any workflow
  • Remaining tools are dynamically selected — based on semantic relevance to the user's query
  • Tool dependencies are enforced — when a tool is selected, its required companion tools are automatically included

Communication Standards

  • Business-appropriate language only (no slang, emoji, or casual tone)
  • Internal IDs are never exposed to users (only user-friendly names)
  • Pixel describes WHAT was created, not HOW (no implementation details in user-facing messages)
  • Next steps are always explained clearly
